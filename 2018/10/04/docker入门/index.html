docker入门(一)

发表于 2018-10-04

基本概念

  • docker image：docker镜像，其中包含有需要运行的文件，镜像用来创建container，一个镜像可以创建成多个container，镜像就像我们下载的iso文件一样，镜像可以在docker hub上下载或者使用docker pull
  • docker container：容器是docker的运行组件，run一个镜像就是启动了一个新的容器，启动以前的容器用start，容器是一个隔离环境，多个容器之间互不影响。
  • docker hub：共享和管理docker镜像，用户可以把自己的diy自己的容器打包成image并且上传到docker hub上。https://registry.hub.docker.com/。

    为什么要使用docker

    容器除了运行其中应用外，基本不消耗额外的系统资源，使得应用的性能很高，同时系统开销尽可能的小。传统虚拟机方式运行十个应用就要起1十个虚拟机，用docker的话只需要启动十个隔离的容器

1.更快速的交付和部署
开发者可以用一个标准的镜像来构建一套开发容器，开发完成之后，运维可以直接使用这个容器来部署代码，docker可以快速创建容器，节约部署时间
2.更高效
docker容器不需要额外的hypervisor支持，它是内核级的虚拟化，因此可以实现更高的效率和性能。
3.更简单
使用docker只需要小小的修改就可以替代以前大量的工作，所有的修改都以增量的方式被分发和更新

docker相对vm的优缺点

优点：vm自身操作系统会占用较多的CPU、内存、硬盘资源。docker只包含应用程序以及依赖库，基于libcontainer运行在宿主机上，并处于一个隔离的环境中。
缺点：docker隔离效果不如vm，共享了宿主机操作系统的一些基础库，网络主要以桥接为主，查看日志不够方便灵活。

安装和使用docker

对应到不同的平台安装过程不太一样，lz使用arch系使用yaourt安装，使用systemctl start docker
